Uses data we already have (`completeness.mappings` + `sufficiency_level`).

Product line: "Veralith tells an honest 'I don't know' apart from a hallucination."
  - **Implementation tiers:** cheap/now (frontend) — detect the rule, show a positive "Honest abstention ✓" badge instead of the red `incomplete_grounded` framing, no library change. Proper (backend/library) — first-class `response_type: answered|abstained` + `abstention_quality: correct|over_refusal`, feeding dashboard + heal suppression; touches the `veralith` eval engine (installed pkg / its source repo), so a deliberate bigger change.

- [x] **[DONE] [BACKEND / RCA] Re-running the same traces spawns duplicate heal cards for one root cause.** Symptom: running the same ~50-trace batch 3–4× opens new cards each time (healed or not); dashboard shows 5 near-duplicate slugs for one problem (`temporal-knowledge-integration`, `real-time-financial-data-integration`, `event-specific-knowledge-gap`, `contextual-retrieval-issues`, `incomplete-context` — all "corpus lacks recent/time-sensitive info").

(D) suppress heals for honest abstentions (see abstention item) — removes most of this noise at source. (E) constrain to a bounded set of canonical categories per failure_cell instead of free-form slugs — structural cap on proliferation. A+B+C = tight low-risk bundle; D = source fix; E = durable structural fix.
